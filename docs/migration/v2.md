# Migration Notes: v2 Direction

These notes describe the current v2-oriented public direction of the project.

## Public Names

- Prefer `Player` as the stable high-level playback entry
- Prefer `Recorder` as the stable high-level recording entry
- Prefer `PlayerRoot` over `RootProvider`

## Audio State Naming

Use:

- `audioState.isStopped`

Compatibility alias still present:

- `audioState.isStoped`

New code should not introduce the deprecated spelling.

## Hook Shape

`useAudioPlayer()` should be treated as the source of:

- top-level control methods
- grouped state under `audioState`

Do not rely on older examples that assume a different top-level structure.

## Import Policy

Always import from:

```tsx
import { Player, Recorder, PlayerRoot, useAudioPlayer, useAudioRecorder } from '@waveform-audio/player';
```

Do not import from internal source paths.

## Recorder Status

Recorder integrations should be built around the public statuses and explicit error codes instead of custom assumptions about `MediaRecorder`.
